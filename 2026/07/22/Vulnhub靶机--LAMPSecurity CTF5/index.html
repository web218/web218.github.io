至此，整个信息收集过程结束，来简单介绍一下陌生端口的作用:

  • ### **25/tcp – SMTP**

简单邮件传输协议，用于**发送邮件**。在此主机上为 Sendmail 2.0.0，攻击者可利用它进行：

- **用户枚举**（VRFY/EXPN 命令）
- **邮件伪造**或**钓鱼**
- 利用历史漏洞（如 Sendmail 的远程代码执行）

------

### **110/tcp – POP3**

邮局协议第3版，用于**接收邮件**。客户端将邮件从服务器下载到本地。此处支持 STARTTLS 但 SSL 证书已过期，可能导致：

- **明文传输凭据**（若未强制加密）
- **中间人攻击**以窃取密码

------

### **111/tcp – rpcbind（也称 portmapper）**

RPC 绑定服务，负责将 **RPC 程序号映射到端口号**。客户端调用 NFS、NIS 等服务时，必须先查询 rpcbind 来获得具体端口。渗透测试中用于：

- 枚举所有注册的 RPC 服务（`rpcinfo -p`）
- 发现不安全的 NFS 导出等突破口

------

### **139/tcp – NetBIOS Session Service**

NetBIOS over TCP/IP，主要用于 Windows 文件和打印机共享，但也出现在 Samba 服务器上。在此通常是 **SMB 的反向兼容**端口。攻击者可能利用它进行：

- 命名管道枚举
- 未认证的共享列表（与 445 配合）

------

### **143/tcp – IMAP**

互联网消息访问协议，用于**在线管理邮件**（邮件存储在服务器上，支持文件夹）。与 POP3 类似，证书过期可能造成明文凭据泄露。其丰富的能力（如 NAMESPACE、THREAD）可能被利用来**提取邮件内容**或**暴力破解邮箱**。

------

### **445/tcp – SMB over TCP**

服务器消息块协议的**直接承载端口**（不经过 NetBIOS），即 Samba 的主要服务端口。此处的 Samba 3.0.26a 具有极高危险性：

- **消息签名被禁用**（中间人攻击、会话劫持）
- 存在著名漏洞 **CVE-2007-4572**（`username map script` 远程代码执行），可直接获得 root shell

------

### **901/tcp – SWAT**

Samba Web 管理工具，通过浏览器管理 Samba 配置。通常运行在 TCP 901 端口，可能存在：

- **弱口令**或默认凭据
- **代码注入**漏洞
- 配置信息的泄露（如共享路径、用户列表）

------

### **44219/tcp – status**

RPC 状态服务（程序号 100024），通常由 **rpc.statd** 提供，用于监视 NFS 客户端的崩溃与恢复状态。该端口是动态注册的，被 rpcbind 映射。攻击者：

- 如果 status 服务存在漏洞（如格式化字符串漏洞），可**远程代码执行**
- 配合 NFS 攻击进行文件锁死等操作
